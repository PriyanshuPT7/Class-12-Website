Database Management System (DBMS) Notes

1. What is Database and DBMS?

Database: A structured collection of data that allows easy access, management, and updating.

DBMS: Software to create, retrieve, update, and manage data. E.g., MySQL, Oracle.

Advantages: Reduces redundancy, ensures integrity, enables data sharing, supports backup, easy querying.

Disadvantages: Expensive, complex, requires trained users, may perform slow for small apps.

3. DBMS Models with Advantages & Disadvantages

Hierarchical Model: Tree structure. Adv: Simple. Disadv: Rigid.

Network Model: Graph structure. Adv: Flexible. Disadv: Complex.

Relational Model: Tables. Adv: Easy to use. Disadv: Slower for large data.

Object-Oriented Model: Objects. Adv: Good for complex data. Disadv: Less supported.

4. Relational Database

Data organized in tables with relationships via keys. Easier, scalable, and uses SQL.

5. Data Redundancy

Definition: Repetition of data across places.

DBMS Solution: Central data, normalization, dependencies.

7. Database Administrator (DBA)

DBA: Person managing database systems.

Responsibilities: Design, security, access control, performance, backups, updates.

9. Keys

Primary Key: Uniquely identifies records. E.g., StudentID

Foreign Key: Refers to primary key in another table. E.g., DeptID

Candidate Key: All possible unique identifiers. E.g., Email, StudentID

10. SQL and Components

SQL: Structured Query Language for managing relational databases.

Components:

  • DDL: CREATE, ALTER, DROP
  • DML: SELECT, INSERT, UPDATE, DELETE
  • DCL: GRANT, REVOKE
  • TCL: COMMIT, ROLLBACK

Common Functions: SUM(), COUNT(), LENGTH(), NOW()

11. Write SQL commands to perform the following operations:-

a. Create a table named students:

CREATE TABLE students (
    Id INT AUTO_INCREMENT PRIMARY KEY,
    Name VARCHAR(100),
    Class VARCHAR(50),
    Marks INT(10)
);

12. Definitions of Key Terms

a. Data Dictionary: A centralized repository of information about data such as meaning, relationships, origin, usage, and format.

b. Primary Key: A field or combination of fields that uniquely identifies each record in a table.

c. Relationship: An association among entities or tables that defines how data in one table relates to data in another.

d. Data Manipulation Language (DML): A subset of SQL used to add, retrieve, and modify data. Examples: SELECT, INSERT, UPDATE, DELETE.

e. Structured Query Language (SQL): A standard programming language used to manage and manipulate relational databases.

f. Data Integrity: The accuracy, consistency, and reliability of data stored in a database.

g. Data Definition Language (DDL): A subset of SQL used to define database schema. Examples: CREATE, ALTER, DROP.

h. Data Security: The protection of data against unauthorized access, corruption, or theft.

i. Database System: A combination of database, DBMS, and associated applications that interact with the database.
