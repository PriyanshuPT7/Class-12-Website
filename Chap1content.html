Database and Information Topics

Information

Definition:
Information refers to processed data that is meaningful and useful for decision-making, problem-solving, and analysis. It provides context to raw data, making it comprehensible and actionable.

Features of Information

  • Accuracy: Information must be correct and free from errors.
  • Relevance: It should be pertinent to the context and needs of the user.
  • Timeliness: Information must be available when needed.
  • Completeness: All necessary data should be included.
  • Consistency: Information should not conflict with other data.

Database and Its Purpose

Definition:
A database is an organized collection of data, typically stored and accessed electronically from a computer system.

Purpose:

  • To store large amounts of data efficiently.
  • To allow quick retrieval and manipulation of data.
  • To ensure data integrity and security.
  • To support multiple users and applications.

Terminologies in Database

  • Table: A collection of related data entries consisting of rows and columns.
  • Field: A single data element; a column in a table.
  • Record: A complete set of fields; a row in a table.
  • Tuple: Another term for a record.
  • Object: An instance of a class in object-oriented databases.
  • Keys:
    • Primary Key: A unique identifier for a record in a table.
    • Foreign Key: A field in one table that uniquely identifies a row of another table.

Data Dictionary

A data dictionary is a centralized repository that defines the structure of the database, including tables, fields, data types, and relationships. It serves as a reference for developers and administrators.

Database Management System (DBMS)

Introduction

A DBMS is software that provides an interface for users and applications to interact with databases, ensuring data is stored, retrieved, and manipulated efficiently.

Objectives

  • To manage data efficiently.
  • To ensure data integrity and security.
  • To provide a user-friendly interface for data operations.

Advantages

  • Data Redundancy Control: Minimizes duplication of data.
  • Data Integrity: Ensures accuracy and consistency.
  • Security: Protects data from unauthorized access.
  • Backup and Recovery: Facilitates data restoration in case of failure.

Disadvantages

  • Complexity: Requires specialized knowledge to manage.
  • Cost: Can be expensive to implement and maintain.
  • Performance: May have overhead due to additional layers of abstraction.

Types of Database Models

1. Hierarchical Model

Data is organized in a tree-like structure where each record has a single parent. Suitable for applications with a clear hierarchical relationship.

2. Network Model

An extension of the hierarchical model, allowing records to have multiple parent records, forming a graph structure. More flexible but complex.

3. Relational Model

Data organized into tables (relations), with relationships established using keys. Most widely used model.

4. Entity-Relationship Model

Uses entities (objects) and relationships to represent data and their connections. Often used in database design.

Integrity Constraints and Types

  • Domain Integrity: Ensures data entered into a field is of the correct type and range.
  • Entity Integrity: Ensures each record has a unique identifier (primary key).
  • Referential Integrity: Ensures foreign keys correctly reference existing records.
  • User-Defined Integrity: Enforces specific business rules.

Normalization

Introduction

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF (First Normal Form): Ensures all columns contain atomic values and records are unique.
  • 2NF (Second Normal Form): Removes partial dependencies; all non-key attributes depend on the whole primary key.
  • 3NF (Third Normal Form): Removes transitive dependencies; non-key attributes are independent of each other.

Advantages

  • Reduces data redundancy.
  • Improves data integrity.
  • Simplifies maintenance.

Disadvantages

  • May lead to complex queries.
  • Can require more joins, impacting performance.

Centralized and Distributed Database

Centralized Database

Data stored at a single location.

  • Advantages: Easier to manage, lower cost.
  • Disadvantages: Single point of failure, can become a bottleneck.

Distributed Database

Data stored across multiple locations.

  • Advantages: Improved performance, availability, scalability.
  • Disadvantages: Complex management, potential consistency issues.

Database Security

Introduction

Database security protects databases from unauthorized access, misuse, or corruption.

Challenges

  • Ensuring data privacy.
  • Protecting against cyberattacks.
  • Managing user access and permissions.

Security Measures

  • Encryption: Protects data confidentiality.
  • Authentication: Verifies user identities.
  • Authorization: Grants appropriate access levels.
  • Auditing: Tracks database activities.

Roles of DBA (Database Administrator)

  • Designing database structure.
  • Implementing and maintaining DBMS.
  • Monitoring performance and security.
  • Backup and recovery of data.

Practical Topics

DDL and DML Languages

  • DDL (Data Definition Language): Defines and manages database structures (e.g., CREATE, ALTER, DROP).
  • DML (Data Manipulation Language): Manages data within those structures (e.g., SELECT, INSERT, UPDATE, DELETE).

SQL Data Types

  • CHAR: Fixed-length character string.
  • VARCHAR: Variable-length character string.
  • BINARY: Fixed-length binary data.
  • VARBINARY: Variable-length binary data.
  • TINYBLOB: Binary large object (max 255 bytes).
  • TINYTEXT: Text string (max 255 characters).
  • TEXT: Text string (max 65,535 characters).
  • LONGTEXT: Text string (max 4,294,967,295 characters).
  • ENUM: Enumeration of predefined string values.
  • BIT: Bit-field type.
  • TINYINT: Very small integer.
  • BOOLEAN: True or false value.
  • INTEGER: Standard integer.
  • FLOAT: Single-precision floating-point number.
  • DOUBLE: Double-precision floating-point number.
  • DECIMAL: Exact numeric data type.
  • DATE: Date in 'YYYY-MM-DD' format.
  • DATETIME: Date and time in 'YYYY-MM-DD HH:MM:SS' format.
